# YMSTUDIO AI Creator OS Use Cases

This page explains who can use YMSTUDIO AI Creator OS and what each person can make with the current local MVP.

## 1. AI Video Beginner

**Who it is for:** Someone who wants to make AI-assisted videos but does not know the order of work.

**Start with:**

- Channel: YMSTUDIO or your channel name
- Topic: AI video production workflow
- Audience: beginner creators
- Goal: show a repeatable production process

**Recommended flow:**

1. Dashboard: press `샘플 프로젝트로 바로 시작`
2. Template Library: choose `일반 영상 훅 설계표`
3. Script Generator: create the first draft
4. AI Shot Planner: split the script into scenes and shots
5. Thumbnail Idea Board: create one thumbnail direction
6. YouTube Calendar: save the upload idea

**Result:** One starter video project with a script draft, shot plan, thumbnail idea, upload memo, asset note, and cost note.

**Why this helps:** The user does not need to start from a blank page or decide which tool comes first.

## 2. YouTube Shorts Creator

**Who it is for:** A creator who posts short tips, AI tool demos, editing tips, or quick tutorials.

**Start with:**

- Topic: one mistake, one tool tip, or one small result
- Audience: people who want a fast answer
- Length: Shorts
- Tone: fast and direct

**Recommended flow:**

1. Template Library: filter by `쇼츠` or `후킹`
2. Script Generator: use the beginner builder with topic, audience, length, and tone
3. Thumbnail Idea Board: create one text hook and image prompt
4. YouTube Calendar: save the upload date and review note

**Result:** A 30-second script, opening hook, scene split, thumbnail text, and upload plan.

**Why this helps:** Shorts are repetitive. The OS turns that repetition into reusable templates.

## 3. ComfyUI / Local AI Workflow User

**Who it is for:** A user who runs image/video workflows locally and needs repeatable notes for models, prompts, settings, and failures.

**Start with:**

- Topic: character consistency, thumbnail batch, product image, or video scene recipe
- Tools: ComfyUI, local models, Claude/ChatGPT for planning
- Goal: avoid losing working settings

**Recommended flow:**

1. Character Consistency Tool: save face, hair, outfit, reference, and negative prompt rules
2. ComfyUI Workflow Manager: save model, resolution, steps, batch, prompt rules, and failure fixes
3. AI Shot Planner: connect workflow notes to specific scenes
4. Creator Asset Manager: save generated file paths and license notes

**Result:** A character bible, ComfyUI recipe card, shot-level prompt notes, and asset records.

**Why this helps:** Local generation often fails because good settings are not documented. This makes the workflow repeatable.

## 4. Tutorial / Course Creator

**Who it is for:** A person making software tutorials, course lessons, or step-by-step explanations.

**Start with:**

- Topic: one lesson or one workflow
- Audience: beginners or students
- Goal: make the lesson easy to follow

**Recommended flow:**

1. Template Library: filter by `튜토리얼`
2. Script Generator: create a lesson script with goal, steps, examples, and CTA
3. AI Shot Planner: turn each step into a screen-recording shot
4. YouTube Calendar: schedule the lesson or series
5. Creator Asset Manager: track lesson files, screenshots, prompts, and references

**Result:** A lesson structure, narration draft, shot checklist, upload plan, and asset list.

**Why this helps:** The creator can explain one idea without forgetting prerequisites, examples, practice steps, or recap.

## 5. Small Business / Product Video

**Who it is for:** A solo operator, small business owner, marketer, or creator selling a template, service, course, or product.

**Start with:**

- Offer: what the product/service helps with
- Audience: who it is for
- Goal: explain the offer clearly
- Tone: trustworthy and practical

**Recommended flow:**

1. Template Library: choose `크리에이터 프로젝트 소개 영상` or `소상공인 설명 영상 템플릿`
2. Script Generator: draft the problem, promise, proof, offer, and CTA
3. Thumbnail Idea Board: create a clear before/after or offer thumbnail
4. YouTube Calendar: plan publish and review timing
5. API Cost Tracker: record paid AI/tool usage if used

**Result:** A product intro script, proof points, CTA, thumbnail direction, and cost note.

**Why this helps:** The message stays focused on the buyer problem instead of becoming a feature list.

## Current MVP Boundary

- The app does not call paid APIs.
- It does not upload files.
- It does not sync across devices.
- It stores data in browser localStorage.
- JSON export/import is the backup path.

This is intentional for the first public MVP: validate whether the workflow is useful before adding accounts, servers, paid APIs, or automation adapters.
