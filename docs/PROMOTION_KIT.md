### Body

```text
I built a public MVP called YMSTUDIO AI Creator OS.

It is a static browser workspace for AI video/content creators. The goal is to help creators avoid blank pages and keep a repeatable workflow before adding paid APIs or automation.

Current flow:
1. Create or sample-fill one Creator Project
2. Pick from 30 creator templates
3. Draft a script from topic/audience/length/tone
4. Send script scenes to a shot planner
5. Create thumbnail directions
6. Save upload calendar notes
7. Track assets and cost notes locally

It does not require login, backend, upload, or paid API calls. Data is stored in browser localStorage and can be exported as JSON.

GitHub:
https://github.com/ymstudio-lab/ymstudio-ai-creator-os

Live demo:
https://ymstudio-lab.github.io/ymstudio-ai-creator-os/outputs/creator-os-dashboard/index.html

I’m looking for practical feedback:
- Does the workflow make sense?
- Which use case fits you?
- Which module would you use first?
- Which template is missing?
- What should connect next?
```

## What Not To Claim Yet

Avoid these until there is real traction or user proof:

- Do not claim it will make money for users.
- Do not claim it replaces professional editors or producers.
- Do not claim automated AI generation.
- Do not claim cloud sync or team collaboration.
- Do not claim privacy beyond what the static local app actually does.

Use this wording instead:

```text
Local-first MVP for organizing AI video workflows and collecting early feedback.
```
