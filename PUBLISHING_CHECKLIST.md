# Publishing Checklist

Use this before updating the public GitHub demo or release.

- Confirm each module opens from its local `index.html`.
- Run module tests.
- Run browser screenshot checks.
- Run browser interaction checks.
- Review `docs/UI_REVIEW.md`.
- Review `docs/V1_PUBLIC_LAUNCH.md`.
- Review README wording for public users.
- Keep private local paths out of screenshots and demo exports.
- Keep privacy and security notes visible before sharing.
- Confirm no API keys, passwords, customer data, payment data, or private contract information is included.
- Confirm the live GitHub Pages demo loads after push.
- Ask early users which screen feels too complex.
